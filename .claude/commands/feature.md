# Feature Implementation

## Feature: $ARGUMENTS

## Scope

This is a **simple website project**. Features are typically:
- New pages (about, products, contact)
- UI components (header, footer, product cards)
- Functionality (contact form, product filtering)
- Styling updates

## Workflow

### 1. Understand the Request
- What page or component is being added/modified?
- How does it fit the business goals?
- What content is needed?

### 2. Check Existing Patterns
- Read similar existing components
- Follow established naming and styling conventions
- Reuse existing UI components where possible

### 3. Implement
- Create/modify files
- Follow Tailwind conventions (mobile-first)
- Ensure responsive design

### 4. Validate

```bash
npm run lint
npm run build
```

### 5. Test Locally

```bash
npm run dev
# Check in browser at localhost:3000
```

## Content Guidelines

When adding website copy:
- Professional tone (selling to hospitals)
- Specific claims (prices, warranty terms)
- Benefits over features
- Short sentences (max 20 words)

## Output

After completion, report:
- Files created/modified
- How to view the changes
- Any content that needs user input (images, specific text)
